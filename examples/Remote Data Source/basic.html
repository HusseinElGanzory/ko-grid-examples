Basic Functionality

Oftentimes entry counts are very large, making it impractical to have all entries in a client side data source. For this purpose, the data-source module offers a ServerSideDataSource. The server side data source only requests data which is required.

Note: How requested data is procured is defined via the strategy pattern. The strategy is given a Query, consisting of a (serializable) predicate, a (serializable) comparator, an offset and limit; how these are translated into a result set is entirely up to you. This example page for instance does not request data from an actual server, but simply computes the result on the client side and returns it with a small delay.
